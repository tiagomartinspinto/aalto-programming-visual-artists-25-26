# Security And Public Content

This repository and the GitHub Pages site are public. Do not commit grades, unpublished third-party work, private feedback, personal data, credentials, private notes, or internal university material.

## If Private Data Is Committed

1. Remove the data from the current tree immediately.
2. Revoke or rotate any exposed token, password, or private URL. Treat it as public.
3. If personal data or unpublished third-party work was exposed, contact the relevant Aalto University channel before republishing.
4. Rewrite Git history only when necessary, then force-push with care and verify the Pages site no longer exposes the content.
5. Open a repository issue or private note for the follow-up work without repeating the sensitive content.

GitHub Pages can keep cached copies briefly. Removing a file from the latest commit is necessary, but not always sufficient for sensitive data.
