SESSION 4 – HANDS-ON EXERCISE
GENERATIVE GRID SYSTEM

GOAL
Build a grid-based generative system using nested loops and Perlin noise.

You will create a structured grid where each element rotates smoothly over time.

---

STEP 1 – RUN THE BASE SYSTEM

1. Create a new Processing sketch.
2. Paste the starter code provided.
3. Run the sketch.
4. Observe what is happening.

Think about:

* What controls the number of tiles?
* What controls the rotation?
* Why does the motion feel smooth?

---

STEP 2 – UNDERSTAND THE STRUCTURE

Locate in the code:

* The nested loops (grid structure)
* The noise() function (variation)
* frameCount (animation over time)
* pushMatrix() and popMatrix() (local transformations)

Make sure you understand these before modifying the sketch.

---

STEP 3 – MODIFY THE SYSTEM

Change at least THREE of the following:

* Increase or decrease gridSize
* Change the noise scale (try 0.005, 0.02, etc.)
* Change rotation speed
* Replace rect() with another shape
* Change shape size
* Add color instead of white
* Add transparency
* Make size depend on noise
* Make rotation depend on mouse position

Experiment. Break things. Adjust numbers.

---

STEP 4 – ADD INTERACTION (OPTIONAL)

Make the system react to the mouse.

Ideas:

* Change opacity based on distance to mouse
* Change size near the cursor
* Change color based on mouseX
* Control rotation speed with mouseY

---

STEP 5 – PUSH IT FURTHER (ADVANCED)

If you finish early:

* Add nested shapes inside each tile
* Combine noise and distance
* Animate both rotation and scale
* Break the grid structure
* Replace noise() with sin()
* Create a composition instead of a uniform pattern

---

REFLECTION

Before finishing, ask yourself:

* What is the rule of my system?
* What variables control the behavior?
* What happens if I change one number?

Small rules create complex visuals.
