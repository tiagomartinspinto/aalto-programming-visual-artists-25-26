HANDS-ON EXERCISE — INTERACTIVE GRID (Processing)

Goal
Create a grid using nested for-loops. Each cell reacts when the mouse hovers over it.

What to do
1) Copy/paste the sketch and run it.
2) Verify:
   - You see a grid of rectangles.
   - When you move the mouse over a cell, it turns red.

Play / Modify (try these one by one)
A) Change the grid size
- Change cols and rows:
  cols = 5; rows = 12;
  cols = 20; rows = 20;

B) Change the hover effect
- Change the hover color:
  fill(0, 200, 255);
- Change the non-hover color:
  fill(30);

C) Add a border (stroke)
- Remove noStroke();
- Add:
  stroke(0);
  strokeWeight(1);

D) Make hovered cells bigger (simple)
- When hovering, draw a smaller/larger rect inside the cell:
  if (hovering) {
    fill(255, 0, 0);
    rect(posX + 4, posY + 4, cellW - 8, cellH - 8);
  } else {
    fill(100);
    rect(posX, posY, cellW, cellH);
  }

E) Use ellipses instead of rectangles
- Replace rect(...) with:
  ellipse(posX + cellW/2, posY + cellH/2, cellW * 0.8, cellH * 0.8);

F) Map color to position (no hover needed)
- Try:
  fill(map(x, 0, cols-1, 0, 255), map(y, 0, rows-1, 0, 255), 150);

G) Add animation
- Make something pulse over time using frameCount:
  float s = 0.7 + 0.3 * sin(frameCount * 0.1);
  ellipse(posX + cellW/2, posY + cellH/2, cellW * s, cellH * s);

Extra Challenge
1) Make the hover affect neighbors (a “radius” effect).
   Hint: use dist(mouseX, mouseY, centerX, centerY)

2) Add a mode switch with keyboard:
   Press '1' = rectangles
   Press '2' = circles
   Press '3' = different colors
