SESSION 5 – HANDS-ON EXERCISE
GENERATIVE SHAPES WITH FUNCTIONS

GOAL
Learn how to create and reuse **functions** to build generative visual compositions.

You will create a system that places **random abstract shapes** across the canvas using a custom function.

---

STEP 1 – RUN THE BASE SYSTEM

1. Create a new Processing sketch.
2. Paste the starter code provided.
3. Run the sketch.
4. Observe the result.

Think about:

* How many shapes are created?
* Where are the shapes placed?
* Why do the shapes look different each time?

---

STEP 2 – UNDERSTAND THE STRUCTURE

Locate in the code:

* The **for loop** that generates multiple shapes
* The **random()** function that controls position, size, and color
* The custom function **drawAbstractShape()**
* The **parameters** passed to the function: `x`, `y`, and `size`

Make sure you understand how the function is being **called multiple times**.

---

STEP 3 – MODIFY THE SYSTEM

Change at least THREE of the following:

* Increase the number of shapes
* Change the range of sizes
* Add **stroke()** to the shapes
* Change the color palette
* Make shapes semi-transparent
* Add a new shape type (line, polygon, etc.)
* Change the background color
* Make shapes larger or smaller

Experiment and observe how small changes affect the composition.

---

STEP 4 – ADD VARIATION (OPTIONAL)

Try adding new behaviour to the shapes.

Ideas:

* Rotate the shapes randomly
* Use **strokeWeight()** to vary line thickness
* Add outlines instead of fills
* Use **mouseX** or **mouseY** to influence size
* Make shapes appear only in part of the screen

---

STEP 5 – PUSH IT FURTHER (ADVANCED)

If you finish early:

* Create a **grid layout** instead of random positions
* Make shapes overlap with **transparent layers**
* Add animation by removing `noLoop()`
* Create **two different drawing functions**
* Combine multiple shape systems

Try to turn the sketch into a **generative artwork**.

---

REFLECTION

Before finishing, ask yourself:

* What does my function control?
* What parameters influence the shapes?
* What happens if I call the function more times?

Functions allow you to **reuse and organize your code**, making complex visuals easier to build.
